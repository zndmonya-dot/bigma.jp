# Bigma 企画書

---

## ■ コンセプト

SNSでネタになっている「言いそうな言葉」を、三段階フォーマットで公式コメントに変換するAIジェネレーター。

「語録文化」「ビッグマウス」を楽しめるエンタメAIツール。

**重要なポイント：**
- 「言いそうな言葉」を創作したネタ
- 三段階フォーマット：本人「〇〇」→通訳「英語」→公式「△△」
- 通訳の時点で既にネタ（実際の通訳ではない）
- 公式も上記に準じてネタにしたもの

---

## ■ サービス概要

* **名称**：Bigma

* **形式**：Webサービス（PC・スマホ対応、PWA対応）

* **入力**：「言いそうな言葉」（創作）1行（最大40文字）

* **出力**：「公式コメント」形式のセリフ（通訳30文字＋公式50文字、合計最大80文字）

  　例：

  　入力：「本当の意味で憧れるのをやめなければ」

  　出力：「憧れは終わった、今こそ俺自身が伝説になる時だ」

* **フォーマット**：

  　本人「本当の意味で憧れるのをやめなければ」

  　通訳「I must stop admiring in the true sense」

  　公式「憧れは終わった、今こそ俺自身が伝説になる時だ」

---

## ■ 特徴

* 🧠 **AIが自動で"公式コメント"を生成**

  * SNSで実際にネタになっている語録パターンを学習
  * 三段階フォーマットに対応
  * 固定テンプレではなく、毎回少し違う表現を出力
  * レート制限によるコスト制御（サーバー側・クライアント側）

* 🎨 **SNSで拡散しやすい1文完結の語録**

  * 生成結果は自動保存（必須）
  * Xで投稿ボタンで直接ツイート可能
  * X投稿時の文字数制限（80文字）に最適化

* ⚾ **野球ファン＋AIユーザーの両層に訴求**

  * MLB選手・日本選手の"語録文化"を活用
  * 打線ランキング（累計上位9位・ベストナイン）で語録を表示（日替わりランダム表示）
  * いいね・リツイート・引用リツイート機能
  * スコア計算：`(いいね数 + 1) × (リツイート数 + 1) × (引用リツイート数 + 1)`

* 📊 **データベースによる精度向上**

  * SNSでネタになっている語録パターンを収集
  * 管理画面（`/admin`）で追加・管理
  * データが増えるほど精度向上

* 🎯 **タブ機能**

  * 新着：最新の語録（ID降順、上位100位まで）
  * 月間：月間ランキング（スコア順、将来的に日付フィルタ実装予定、上位100位まで）
  * 累計：スコア順ランキング（上位100位まで）

## ■ 技術構成

* **フロントエンド**：Next.js 16 / React 19 / TypeScript

## ■ データ収集について

### 収集すべきフォーマット

```
本人「〇〇」
通訳「英語」
公式「△△」
```

### 重要なポイント

1. **「（山本由伸）」は付けない**
   - 公式の最後に「（山本由伸）」は不要

2. **三段階フォーマット**
   - 本人「〇〇」: 「言いそうな言葉」（創作）
   - 通訳「英語」: ハリウッド通訳者（園田さん）が大袈裟に翻訳したもの（実際の通訳とは違って大袈裟に翻訳されたもの、公式の元となる翻訳）
   - 公式「△△」: その大袈裟な通訳を聞いて感化を受けた公式が発信する公式コメント（通訳の内容が大袈裟なので、それを元にした公式コメントも大袈裟になる）
   - **通訳者の背景**: 園田さんはかつてハリウッドの世界で翻訳家を務めていた経験を持ち、シンプルで何気ない一言でも、まるで熱い映画のワンシーンのような臨場感とドラマチックさを与えるタイプの通訳を行います。本人は意図していませんが、結果的にそのような通訳になります。
   - **公式コメントの生成プロセス**: 本人は園田さんの通訳を知りません。しかし、公式（公式コメントを発信する側）が園田さんの通訳を聞いて「おおー！」となり、そのドラマチックな通訳に触発されて公式コメントを発信します。

3. **実際のネタを収集**
   - AIが生成した例ではなく、SNSで実際に拡散されているネタを収集することが重要

### データ収集先

* **X（Twitter）**: 「山本由伸 語録」「山本由伸 ネタ」「山本由伸 意訳」で検索
* **まとめサイト**: スポーツまとめサイト、野球ファンブログ
* **YouTube**: 語録まとめ動画、ネタ動画

### 精度向上の重要性

- データが増えるほど、AI生成の精度が向上
- SNSで実際に拡散されているネタパターンを収集することが重要
- 管理画面（`/admin`）で簡単に追加・管理可能

詳しくは [DATA_COLLECTION.md](./DATA_COLLECTION.md) を参照してください。

レート制限に達した場合は、エラーメッセージを表示します。

## ■ スコア計算

打線ランキングや累計ランキングでは、以下の式でスコアを計算します：

```
スコア = (いいね数 + 1) × (リツイート数 + 1) × (引用リツイート数 + 1)
```

## ■ 注意事項

* **実際の発言ではない**: 本人「〇〇」は「言いそうな言葉」を創作したネタです

* **通訳もネタ**: 通訳「英語」はハリウッド通訳者（園田さん）が大袈裟に翻訳したもの（実際の通訳とは違って大袈裟に翻訳されたもの、公式の元となる翻訳）。園田さんはかつてハリウッドの世界で翻訳家を務めていた経験を持ち、シンプルで何気ない一言でも、まるで熱い映画のワンシーンのような臨場感とドラマチックさを与えるタイプの通訳を行います。

* **公式コメントの生成プロセス**: 本人は園田さんの通訳を知りません。しかし、公式（公式コメントを発信する側）が園田さんの通訳を聞いて「おおー！」となり、そのドラマチックな通訳に触発されて公式コメントを発信します。

* **データ収集が重要**: SNSで実際に拡散されているネタパターンを収集して、プロンプトの精度を上げる必要があります

* **レート制限**: サーバー側（本番環境のみ）・クライアント側でコスト制御を行っています

* **不適切な内容のフィルタリング**: ChatGPTが不適切な表現を自動的にフィルタリングします

* **文字数制限**: 入力40文字、出力（通訳30文字+公式50文字=合計80文字）に最適化されています
